📌 Viết hàm lấy Ward theo District
const getWardsByDistrict = (cityCode, districtCode) => {
    const city = addressData.find(c => c.code === Number(cityCode));
    if (!city) return [];

    const district = city.districts.find(d => d.code === Number(districtCode));
    return district ? district.wards : [];
};

giải thích khi nào giá trị của useRef thay đổi , và tại sao không dùng useState 

1. Khi nào giá trị của useRef thay đổi?

useRef tạo ra một object cố định, có dạng:

const searchRef = useRef(null);


Nó tạo ra object:

{ current: null }

Giá trị của useRef thay đổi khi:
👉 React gắn element DOM vào ref

Khi component render xong, React sẽ gán:

searchRef.current = <div className="search" ... >


=> Tức là current sẽ chứa DOM node của phần tử <div className="search">.

Nếu bạn console.log trong render, giá trị ban đầu là null → sau khi render xong, React mới gán vào.

👉 Bạn thay đổi thủ công

Bạn có thể chỉnh:

searchRef.current = 123;


Nhưng React sẽ không trigger re-render.

❗ Quan trọng:
useRef không gây re-render khi thay đổi → đây là điểm khác biệt lớn nhất với useState.
✅ 2. Tại sao trong đoạn code này phải dùng useRef thay vì useState?

Trong đoạn code:

const searchRef = useRef(null);

Bạn dùng ref để:

➤ Lấy DOM element của search box:
if (searchRef.current && !searchRef.current.contains(event.target)) {
    setShowDropdown(false);
}


Bạn cần kiểm tra xem người dùng click ra ngoài search box hay không.

🧠 Vì sao không dùng useState?
❌ useState không chứa DOM element

State chỉ chứa dữ liệu để render UI, không chứa được:

DOM node

object không serializable

Bạn không thể viết:

const [searchRef, setSearchRef] = useState(null);


Rồi gán:

<div ref={setSearchRef}>


→ Về lý thuyết có thể làm, nhưng nó sẽ gây re-render mỗi lần ref thay đổi, điều này không cần thiết và gây lỗi.

⚠ Vì sao dùng useState ở đây là sai?

DOM node không nên đi qua bộ render của React
React không designed để state chứa các DOM element.

Nếu dùng state, mỗi lần ref cập nhật → component re-render → React lại gán ref → re-render → loop
Gây re-render thừa hoặc loop vô hạn.

useRef giữ giá trị ổn định qua mỗi lần render, trong khi state thì không.

🧩 Tóm lại:
So sánh	useRef	useState
Thay đổi có gây re-render?	❌ Không	✅ Có
Lưu DOM element?	✅ Có	❌ Không nên
Dùng để đọc/ghi giá trị tức thời?	✅ Có	❌ Không (asynchronous)
Dùng để làm UI cập nhật?	❌ Không	✅ Có
🎉 Kết luận
Bạn bắt buộc dùng useRef cho trường hợp:

✔ muốn lấy phần tử DOM
✔ muốn lưu giá trị không cần render lại
✔ muốn dùng trong event listener (click outside)

Nếu dùng useState thì:

Không chứa được DOM

Mỗi thay đổi gây re-render không cần thiết

Có thể gây bug click-outside

1. Hai key trong component của bạn là:
✅ Key 1 — dùng cho grid
<div className={`deals-grid slide-${direction}`} key={currentPage}>

✅ Key 2 — dùng cho từng item
<Link to={`/product/${deal.id}`} key={deal.id} className="deal-card">


Hai key này phục vụ 2 mục đích hoàn toàn khác nhau.

🧩 2. Key của deals-grid (key = currentPage)
❗ Key này dùng để ép React re-create lại toàn bộ grid khi trang thay đổi.

Ở đây key={currentPage} làm React hiểu rằng:

"Mỗi khi currentPage đổi → component này là một component hoàn toàn mới → phải unmount + mount lại."

Điều này cực kỳ quan trọng nếu bạn đang dùng hiệu ứng slide CSS (slide-left / slide-right).

Nếu không có key={currentPage}, vấn đề xảy ra:

React chỉ update lại danh sách item

Nhưng không trigger re-render hoàn toàn

→ animation slide có thể không chạy hoặc chạy lỗi

→ React sẽ chỉ cập nhật phần tử bên trong, không reset animation

Vì sao key hoạt động?

Khi key khác nhau:

React behavior:

old key: 0 → new key: 1  
=> React unmount deals-grid cũ  
=> React mount deals-grid mới  
=> CSS animation chạy lại chuẩn

🧩 3. Key cho từng deal item (key = deal.id)

Dùng khi render list:

{deals.map((deal) => (
  <Link key={deal.id}>

Mục đích:

Giúp React nhận biết từng phần tử trong danh sách để tối ưu diffing.

Khi dữ liệu thay đổi React sẽ biết:

item nào giữ nguyên

item nào thay đổi

item nào cần thêm/xoá

Nếu không có key → React sẽ:

cảnh báo

render sai item

gây lỗi animation / lỗi state bên trong mỗi item

📌 Tóm tắt sự khác nhau
Key	Mục đích	Placement	Cách hoạt động
key={currentPage}	Reset toàn bộ grid khi đổi trang → cho hiệu ứng slide	Trên <div className="deals-grid">	Mỗi trang là 1 component mới
key={deal.id}	Giúp React nhận biết item trong mảng, tối ưu render	Trong .map()	Giữ item ổn định, tránh render lỗi
🎯 Ví dụ dễ hiểu
➤ Key cho grid

Giống như thay toàn bộ trang → cần "lật sang trang khác"

➤ Key cho item

Giống như đánh số ghế → React biết ghế nào thay đổi, ghế nào giữ nguyên.


Định nghĩa Key trong React là gì?

Key là một “định danh duy nhất” mà React dùng để nhận biết từng phần tử trong danh sách hoặc từng phiên bản của một component.

Nói cách khác:

Key giúp React biết phần tử nào giữ nguyên, phần tử nào thay đổi, phần tử nào bị xoá hay cần tạo mới.

🧠 Tại sao React cần key?

React không so sánh UI theo từng phần tử HTML, mà so sánh theo tree (cây).

Khi danh sách thay đổi, React cần biết:

Item nào là item cũ

Item nào mới xuất hiện

Item nào mất đi

➡️ Nếu không có key, React không biết đâu là đâu → dễ render sai.

📌 Ví dụ cực dễ hiểu

Giả sử bạn có 3 item:

A, B, C


Và bạn thêm item “X” vào đầu:

X, A, B, C

🔹 Nếu có key (unique):

Thứ tự thay đổi nhưng React biết:

A vẫn là A

B vẫn là B

C vẫn là C

X là phần tử mới

➡️ React chỉ thêm “X”.

🔹 Nếu không có key:

React không biết đâu là đâu → nó đoán:

X là A

A là B

B là C

C biến mất

➡️ Các component con có state sẽ bị lẫn, UI nhảy loạn.

🎯 Khi nào key được React dùng?

React dùng key để:

1️⃣ So khớp (reconcile) danh sách .map()
items.map(item => <div key={item.id}>{item.name}</div>)

2️⃣ Ép unmount + remount component (reset hoàn toàn)
<div key={currentPage}> ... </div>

3️⃣ Quản lý animation và transition
4️⃣ Giữ state ổn định trong từng item
📌 Key phải thoả điều kiện gì?
Thuộc tính	Yêu cầu
unique	Trong cùng 1 danh sách phải khác nhau
stable	Không được thay đổi giữa các lần render
predictable	Không dùng số index nếu danh sách thay đổi thứ tự
🚫 Lỗi phổ biến: Dùng index làm key
key={index}


→ Không nên dùng nếu danh sách có thêm/xoá/sắp xếp lại.

🧩 Định nghĩa đầy đủ, chuẩn React:

Key là thuộc tính đặc biệt giúp React xác định xem một element trong danh sách có thay đổi giữa các lần render hay không, từ đó tối ưu quá trình diffing và rendering.

🖱 4. Click ra ngoài thì ẩn dropdown
useEffect(() => {
  const handleClickOutside = (event) => {
    if (searchRef.current && !searchRef.current.contains(event.target)) {
      setShowDropdown(false);
    }
  };
  document.addEventListener("mousedown", handleClickOutside);
  return () => document.removeEventListener("mousedown", handleClickOutside);
}, []);


Giải thích:

Khi click chuột xuống bất kỳ đâu (mousedown),

Nếu click không nằm trong vùng searchRef (tức là ngoài ô tìm kiếm),

Thì setShowDropdown(false) → đóng dropdown.

Cuối cùng, hàm return dọn dẹp listener khi component bị unmount.

Nếu prev > 0: tức là không phải trang đầu tiên, thì lùi lại 1 trang (prev - 1).

Nếu prev === 0: tức là đang ở trang đầu, thì vòng lại trang cuối cùng (totalPages - 1).

setCurrentPage((prev) => (prev < totalPages - 1 ? prev + 1 : 0));

Nếu prev < totalPages - 1: tức là chưa đến trang cuối, thì tiến thêm 1 (prev + 1).

Nếu prev === totalPages - 1: tức là đang ở trang cuối, thì vòng lại trang đầu tiên (0).
